Introduction to Flex/Lex


  • What is lex/flex
    • lex is a scanner generator.
    • Flex: an open source implementation of the original UNIX lex utility.
  • Why

How it works


  • lex is a scanner generator.
  • Input ia a set of regular expressions and associated actions written in c/c++
  • Output is a table-driven scanner (lex.yy.c)

flex input


  • First part
  • %%
  • patern {action}
  • ......
  • %%
  • Third part

Running lex code


  • $ flex scan.l
  • $ cc lec.yy.c
  • $ ./a.out

Bisson/Yacc


  • Yacc: Yet Another Compiler Compiler
    • Parses and does semantic processing on the stream of tokens produced by lex.
  • Bison: GNU parser parser, upward compatibility with yacc.

First part of a yacc specification


  • C/C++ declaration enclosed in %{%}
  • yacc definations
    • %start
    • %token
    • %union

yacc - productions


  • The middle section represents a grammar - a set of productions.
  • Multiple right-hand sides may follow seperated by a "|"
  • Actions associated with a rule are entered in braces.

Running flex and bison together

  • $ bison -d grammar.y
  • $ flex scan.l
  • $ g++ grammar.tab.c lex.yy.c -lfl
  • $ ./a.out
